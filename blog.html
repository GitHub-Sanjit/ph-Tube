Discuss the scope of var, let, and const!

VAR :

  • Scope : Variables declared with var are function-scoped. This means that their scope is limited to the function in which they are declared.
  • Hoisting : var is hoisted to the top of its scope, which means that you can use the variable before it's declared, but it will be undefined until the declaration is encountered in the code.
  • Reassignment : var allows for redeclaration within the same scope.

LET :

  • Block Scope : Variables declared with let are block-scoped. This means that their scope is limited to the block (enclosed by curly braces) in which they are declared.
  • Hoisting : let is hoisted to the top of its block, but unlike var, it is not initialized until the actual declaration is encountered.
  • Reassignment : let does not allow for redeclaration within the same scope, but it allows for reassignment.

CONST :

  • Block Scope : Variables declared with const are also block-scoped.
  • Hoisting : Like let, const is hoisted to the top of its block but not initialized until the actual declaration is encountered.
  • Reassignment : const variables must be initialized at the time of declaration, and their values cannot be reassigned.
Tell us the use cases of null and undefined!

null :

  • Intentional Absence of value : It is a deliberate assignment to a variable or an object property to signify that it has no value or is explicitly empty.
  • Use cases :
    • Explicitly Absence of values : null is often used to explicitly indicate that a variable or object property has no assigned value or that a function returns no meaningful result.
    • Resetting Variable Values : Setting a variable to null can be a way to reset its value or indicate that it should not be used in its current state.

undefined :

  • Default Value for Uninitialized Variables: Variables that are declared but not assigned a value are automatically initialized with undefined.
  • Use cases :
    • Object Properties that are not Defined: Accessing an object property that is not defined results in undefined.
    • Function Parameters that are not Provided: If a function is called with fewer arguments than the number of parameters it expects, the missing parameters are set to undefined.
    • Return Value of Functions with No Explicit Return: If a function does not explicitly return a value, it implicitly returns undefined.
What do you mean by REST API?

REST API :

A REST API (Application Programming Interface) is an implementation of the principles of REST that provides a set of rules for building web services. RESTful APIs enable communication and data exchange between different software systems over the internet.
Resources are identified by URIs (Uniform Resource Identifiers), and they can be manipulated using standard HTTP methods (GET, POST, PUT, DELETE). Resources represent entities such as objects, data, or services.
Commonly used HTTP methods and status codes are integral to the functioning of RESTful APIs.
